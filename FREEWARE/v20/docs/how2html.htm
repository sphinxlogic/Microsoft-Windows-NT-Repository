Basic page structure

The basic page coding includes the <html>, <head>, <title>, and <body> tags.

Use of tags:

  • The <html> and </html> tags enclose the start and end of the HTML file. They must be the first and last tags in the file.
  • The <head> and </head> tags enclose the HTML file header information. The header information contains the title tags.
  • The <title> and </title> tags enclose a text string that identifies the HTML page to the reader. The browser displays this title information in a nonscrollable area of the browser.
  • The <body> and </body> tags enclose the main information of the page.

Example: 


<html>

<head>
<title>Sample Page and HTML Examples </title>
</head>

<body>
..... text of the document
</body>

</html>

Heads (or Heading Levels)

HTML supports up to six levels of headings. The <h1>, <h2>, <h3>, <h4>, <h5>, and <h6> tags indicate the level of heading; each heading has its own appearance on a browser.

Example: 


<h1>This is a Level 1 Head </h1>

<h2>This is a Level 2 Head </h2>

<h3>This is a Level 3 Head </h3>

<h4>This is a Level 4 Head </h4>

Result:

This is a Level 1 Head

This is a Level 2 Head

This is a Level 3 Head

This is a Level 4 Head

Paragraphs

The <p> and </p> tags enclose each paragraph of information in the HTML file. The paragraph tags should not enclose heads, lists, or preformatted examples.

Example: 


<p>
This is a paragraph of text. 
This is a paragraph of text. 
This is a paragraph of text. 
This is a paragraph of text. 
</p>

Result:

This is a paragraph of text. This is a paragraph of text. This is a paragraph of text. This is a paragraph of text.

Ordered (numbered) and Unordered (bulleted) Lists

The list coding includes the <ol> and </op> ("ordered" or numbered list) or <ul> and </ul> ("unordered" or bulleted list) tags, and the <li> and </li> tags.

Use of tags:

  • The <ol> and </ol> or <ul> and </ul> tags enclose the start and end of a list of items. You can nest lists up to six levels deep.
  • The <li> and </li> tags enclose the text of the list item. The list item can contain paragraph tags.

Example: 


<ul>
<li>This is unordered list item one. </li>
<li>This is unordered list item two. </li>
<li>This is unordered list item three. 
   <ol>
   <li>This is ordered sublist item one. </li>
   <li>This is ordered sublist item two. </li>
   <li>This is ordered sublist item three. </li>
   </ol>
</li>
<li>This is unordered list item four. </li>
</ul>

Result:

  • This is unordered list item one.
  • This is unordered list item two.
  • This is unordered list item three.
    1. This is ordered sublist item one.
    2. This is ordered sublist item two.
    3. This is ordered sublist item three.
  • This is unordered list item four.

Emphasized text

The emphasis coding includes the bold, italic, code example, and horizontal rule tags.

Use of tags:

  • The <b> and </b> tags enclose a text string to make it bold. The string cannot contain any higher-level tags like heads or lists.
  • The <i> and </i> tags enclose a text string to make it italic. The string cannot contain any higher-level tags like heads or lists.
  • The <pre> and </pre> (preformatted) tags enclose a text string or one or more lines of code. The marked section cannot contain any higher-level tags like heads or lists.
  • The <hr> tag is a standalone tag. It cannot be combined with another tag; it will always force a line break.

Example: 


This line has a <b>bold </b> word.

This line has an <i>italicized </i> word.

This line introduces preformatted text:
<pre>
  Line 1
  Line 2
  Line 3
</pre>

This line has is followed by a horizontal rule:
<hr>

Inline Graphics

The inline graphics coding has several parts, including img src and alt.

Syntax:

<img src="FILENAME.GIF" alt="Description of art">

Parts:

  • img src="FILENAME.GIF" defines the name of the graphic file to display. This filename is case-sensitive on some platforms.
  • alt="string" specifies a text string that is displayed on non-graphic viewers.

Example: 

<img src="DIGITAL.GIF" alt="[Digital logo]">

Popup Graphics

The popup graphics coding has several parts, including a, href, img src, and alt.

Syntax:

<a href="POPGRAPH.GIF"><img src="HOTGRAPH.GIF" alt="Description of art"></a>

Parts:

  • a is an anchor tag. Click here for more information about anchors.
  • href="POPGRAPH.GIF defines the name of the graphic file that will be popped up. This filename is case-sensitive on some platforms.
  • img src="HOTGRAPH.GIF" defines the name of the hot graphic file. This filename is case-sensitive on some platforms.
  • alt="string" specifies a text string that is displayed on non-graphic viewers.

Example: 

<a href="DIGITAL.GIF"><img src="T_NAME.GIF" alt="Description of art"></a>

Anchors

The anchor tag has two uses:

  1. When used with a NAME="" value, it defines (or sets an anchor to) a specific location in a file.
  2. When used with an HREF="" value, it jumps back to an anchored location in a file. For more information about anchors and HREF values, click here (for jumping within a file) or click here (for jumping to another file).

Syntax:

<a name="ANCHOR_NAME">Anchor text</a>

Parts:

  • The a tags enclose the start and end of the text string to which you anchor the symbol.
  • name="ANCHOR_NAME" specifies the symbolic name you are associating with the anchor text. The anchor name is case-sensitive on some platforms.
  • Anchor text is the text to which you are linking (or anchoring) the symbol or anchor name. The anchor text must contain at least one character. Empty anchors are not allowed in HTML.

Example: 


<h2><a name="ANCHORS">Anchors</a></h2>

Hypertext Jump Within a File

When you use anchor tags with an HREF="#ANCHOR_NAME" value, HTML jumps back to an anchored location in the current file. The a tags enclose the start and end of the text string you want to make hot. Clicking on the hot text causes the system to jump to the matching symbol string.

Syntax:

<a href="#ANCHOR_NAME">Hot text</a>

Parts:

  • The a tags enclose the start and end of the text string to which you anchor the symbol.
  • href="#ANCHOR_NAME" specifies the symbolic name you want to jump to. The pound sign is a required character. The anchor name is case-sensitive on some platforms.
  • Hot text is the text that will appear as hot on the Web page. The hot text must contain at least one character. Empty hot text will not be selectable.

Example: 


For more information about anchors and HREF values, 
<a href="#JUMP_IN_FILE">click here</a> (for jumping within a file) or 
<a href="#JUMP_TO_FILE">click here</a> (for jumping to another file).

Hypertext Jump to Another File

When you use anchor tags with an HREF="FILENAME.HTML#ANCHOR_NAME" value, HTML jumps back to an anchored location in a different file. The a tags enclose the start and end of the text string you want to make hot. Clicking on the hot text causes the system to jump to the matching symbol string.

Syntax:

<a href="FILENAME.HTML#ANCHOR_NAME">Hot text</a>

Parts:

  • The a tags enclose the start and end of the text string to which you anchor the symbol.
  • href="FILENAME.HTML#ANCHOR_NAME" specifies the filename and symbolic name you want to jump to. The filename is case-sensitive on some platforms. The pound sign is a required character. The anchor name is case-sensitive on some platforms.
  • Hot text is the text that will appear as hot on the Web page. The hot text must contain at least one character. Empty hot text will not be selectable.

Example: 


For more information about anchors and HREF values, 
<a href="HTML_EXM.HTML#JUMP_IN_FILE">click here</a> (for jumping within a file) or 
<a href="HTML_EXM.HTML#JUMP_TO_FILE">click here</a> (for jumping to another file).
